## 2019 Overview: The Year of Intcode

2019 is famous for being "The Year of Intcode" - a virtual machine that gets incrementally built and reused throughout the month. Many problems after Day 9 assume you have a working Intcode interpreter. The key is to build a solid, reusable Intcode module early.

## Intcode Guide

Since many 2019 problems use Intcode, here's a summary of all features needed:

### Opcodes
| Opcode | Name | Parameters | Description |
|--------|------|------------|-------------|
| 1 | ADD | 3 | p3 = p1 + p2 |
| 2 | MUL | 3 | p3 = p1 Ã— p2 |
| 3 | INPUT | 1 | p1 = input |
| 4 | OUTPUT | 1 | output p1 |
| 5 | JNZ | 2 | if p1 â‰  0, ip = p2 |
| 6 | JZ | 2 | if p1 = 0, ip = p2 |
| 7 | LT | 3 | p3 = p1 < p2 ? 1 : 0 |
| 8 | EQ | 3 | p3 = p1 = p2 ? 1 : 0 |
| 9 | REL | 1 | relative_base += p1 |
| 99 | HALT | 0 | stop |

### Parameter Modes
| Mode | Name | Meaning |
|------|------|---------|
| 0 | Position | Value at address |
| 1 | Immediate | Value directly |
| 2 | Relative | Value at (relative_base + value) |

### Implementation Tips
- Store memory as a Map for sparse access beyond initial program
- Return VM state for suspension/resumption (important for Day 7, 11, 13, etc.)
- Support arbitrarily large integers (native in Elixir)
- Make input/output flexible (queues, callbacks, etc.)
